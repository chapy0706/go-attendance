# 0001 Initial Product Spec

## 背景

勤怠打刻アプリを題材に、Go + PostgreSQL + Next.js による業務アプリケーションを作成する。

このプロジェクトでは、フレームワークに処理を隠しすぎず、設計・責務分離・DBトランザクション・認証認可・バッチ処理を理解することを重視する。

## 目的

- 出勤・退勤・休憩を記録できる
- ユーザーが自分の勤怠を確認できる
- 管理者がユーザーの勤怠を確認できる
- 将来的に修正申請・承認・月次締め・CSV出力を追加できる構造にする

## MVPスコープ

### 対象

- 一般ユーザー
- 管理者

### 機能

- ログイン
- 出勤打刻
- 退勤打刻
- 休憩開始
- 休憩終了
- 自分の勤怠一覧
- 管理者の勤怠一覧

## MVPでは扱わないもの

- 給与計算
- シフト管理
- 複雑な勤務体系
- 有給管理
- 外部勤怠システム連携

## 業務ルール初期案

- 出勤前に退勤はできない
- 出勤中に再度出勤はできない
- 退勤後に休憩開始はできない
- 休憩中に退勤はできない
- 休憩開始後、休憩終了前に再度休憩開始はできない
- 同一日の勤怠記録を基本単位とする
- 時刻はサーバ側で記録する

## 非機能要件

- 認証済みユーザーのみAPIを利用できる
- 管理者用APIは管理者権限を必須にする
- DB更新はトランザクションで保護する
- 主要な業務ルールはテストで保護する
- 本番環境でヘルスチェックAPIを提供する

## 未決事項

- 認証方式
- 勤怠日の区切り時刻
- タイムゾーンの扱い
- 管理者ロールの付与方法
- 打刻修正申請の詳細仕様
- バッチ実行方式
